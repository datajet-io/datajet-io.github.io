Searching ¶

The datajet search engine is able to deliver highly relevant, faceted search results in real-time. It provides capabilities for full-text search as well as extensive options of customizing search results to your specific needs for web and mobile applications.

Using keywords ¶

A typical search use case is to search using keywords entered in a search box by the user.

Such a use case translates into the following search request on datajet:

curl -XPOST "https://hawk.<region>.datajet.io/2.0/search/?key=<authKey>" -d '{

      "q": "foo"
}'

Filtering ¶

You limit search result to only include items that match specific critieria using filters. Filtering is supported on the following fields: brand, category, attributes and price.

The following example request returns a list of all items that match the these criteria:

  • The brand name matches Foo or Bar

  • The price is higher than 30

  • The category is Apparel

curl -XPOST "https://hawk.<region>.datajet.io/2.0/search/?key=<authKey>" -d '{

      "filters": {
        "brand.name": [
          "Foo",
          "Bar"
        ],
        "price": {
          "range": {
            "lte": 30
          }
        },
        "categories": "Apparel"
      }
}

Searching ¶

The datajet API supports searching using POST or GET HTTP requests. POST requests are made against the /search endpoint, while GET requests are made against the /product endpoint.

Search using POST  ¶

Two features are only available with POST-requests:

  • Geo-search: Searching using latitude / longitude

  • Numeric facetting: Faceting using numeric values

Search using GET  ¶

/product
GET/product{?key,q,page,size,sort,fields,facet,facet_count,category_dept,facet_include_filtered,filter,dum,dum_count,top_product_ids}

Example URI

GET https://hawk.datajet.io/product?key=&q=foo&page=2&size=3&sort=price_desc&fields=id,title&facet=attribute:sale&facet_count=10&category_dept=11&facet_include_filtered=true&filter=attribute:sale&dum=&dum_count=&top_product_ids=
URI Parameters

key
string (required) 

Authentication key

q
string (optional) Example: foo

Keywords to search for

page
number (optional) Default: 1 Example: 2

Request specific page from the search results, maximum value is 100.

size
number (optional) Default: 10 Example: 3

Number of items to include in the search result

sort
string (optional) Default: relevance Example: price_desc

Sort items in search result

Choices: relevance price_desc price_asc discount_desc discount_asc time_asc time_desc

fields
string (optional) Example: id,title

Fields to include in the response, e.g. price, title, id. If the parameter is not set all fields are returned.

filter
string (optional) Example: attribute:sale

Include only items that match the specified criteria. To use multiple criteria include as many filter parameter as necessary. Use price_min to set a minimum threshold on the price, e.g. price_min:200

facet
string (optional) Example: attribute:sale

Include only items that match the specified criteria. To use multiple criteria include as many filter parameter as necessary. Use price_min to set a minimum threshold on the price, e.g. price_min:200

facet_count
number (optional) Default: 0 Example: 10

Limits the number of different occurrences when faceting, e.g., 20 will only return the 20 most frequent products for the specified facets

facet_include_filtered
string (optional) Default: false Example: true

Will include an item even if it would be excluded according to facet_count

category_dept
number (optional) Default: 0 Example: 11

Specifies the depth of the category tree when faceting.

dum
string (optional) Default: replace 

Did-you-mean / Spellchecking

Choices: replace merge info

dum_count
number (optional) Default: 0 

Threshold for number of item when considering Did-you-mean.

top_product_ids
string (optional) 

Put the provided products on top of the search results, the array order will be preserved in the search result.
